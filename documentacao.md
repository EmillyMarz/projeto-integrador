## 📌 Definição do Problema
Muitos estudantes enfrentam dificuldades para organizar suas tarefas, trabalhos e provas, o que resulta em atrasos, acúmulo de atividades e baixo desempenho acadêmico.

Grande parte utiliza métodos desorganizados, como anotações em diferentes lugares, o que dificulta o controle das atividades.

---

## 👥 Público-alvo
Estudantes do ensino médio e superior que possuem dificuldades em organizar suas atividades acadêmicas.

---

## 🌍 Contexto Social
O projeto busca auxiliar estudantes na organização de suas rotinas, reduzindo estresse, ansiedade e melhorando o desempenho acadêmico.

## 📋 Requisitos Funcionais (RF)

- **RF01 – Cadastro de Usuário**  
O sistema deve permitir que o usuário realize seu cadastro informando dados básicos (nome, e-mail e senha).

- **RF02 – Login de Usuário**  
O sistema deve permitir que usuários autenticados façam login utilizando e-mail e senha.

- **RF03 – Gerenciamento de Tarefas**  
O sistema deve permitir que o usuário crie, edite e exclua tarefas.

- **RF04 – Notificações de Atividades**  
O sistema deve enviar notificações sobre prazos e atualizações de tarefas.

- **RF05 – Visualização de Tarefas**  
O sistema deve permitir que o usuário visualize suas tarefas organizadas (por status, data ou prioridade).

- **RF06 – Recuperação de Senha**  
O sistema deve permitir que o usuário recupere sua senha através do e-mail cadastrado.

- **RF07 – Perfil do Usuário**  
O sistema deve permitir que o usuário visualize e edite suas informações pessoais (nome, e-mail e senha).

- **RF08 – Definição de Prioridade de Tarefas**  
O sistema deve permitir que o usuário defina níveis de prioridade (baixa, média, alta) para cada tarefa.

- **RF09 – Histórico de Atividades**  
O sistema deve registrar e permitir que o usuário visualize o histórico de ações realizadas nas tarefas.

- **RF10 – Compartilhamento de Tarefas**  
O sistema deve permitir que o usuário compartilhe tarefas com outros usuários.

---

## ⚙️ Requisitos Não Funcionais (RNF)

- **RNF01 – Desempenho**  
O sistema deve suportar diversos usuários simultaneamente sem comprometer o desempenho.  
**Justificativa:** Permitir que vários usuários utilizem o sistema ao mesmo tempo sem travamentos.

- **RNF02 – Segurança**  
O sistema deve garantir a segurança dos dados dos usuários.  
**Justificativa:** Evitar vazamento de informações pessoais.

- **RNF03 – Disponibilidade**  
O sistema deve estar disponível 24 horas por dia, 7 dias por semana.  
**Justificativa:** Garantir acesso contínuo ao sistema.

- **RNF04 – Responsividade**  
O sistema deve funcionar em dispositivos móveis.  
**Justificativa:** Permitir acesso em qualquer lugar.

- **RNF05 – Usabilidade**  
O sistema deve possuir uma interface intuitiva.  
**Justificativa:** Facilitar o uso por novos usuários.

## 👤 User Stories

- Como usuário, quero me cadastrar no sistema para acessar minhas tarefas.

- Como usuário, quero fazer login para acessar minha conta com segurança.

- Como aluno, quero criar tarefas para organizar meus estudos.

- Como usuário, quero definir prioridade nas tarefas para saber o que fazer primeiro.

- Como usuário, quero visualizar minhas tarefas para acompanhar minhas atividades.
